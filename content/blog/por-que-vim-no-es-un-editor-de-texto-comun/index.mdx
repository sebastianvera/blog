Quizás te has preguntado cuál es la diferencia entre ocupar [Vim] o [Sublime Text],
[Atom], [VS Code], etc. La principal diferencia es que Vim tiene como foco ser un
editor de texto que aumenta la productividad en su propósito más básico,
["editar y crear cualquier tipo de texto de manera eficiente"](https://vim.org).

La idea de este post es dar a conocer algunas de las características más básicas
de este editor y motivarte a ocupar esta gran herramienta, que en mi caso, ha
significado un gran incremento de productividad al momento editar texto o
programar.

> Vim además de ser un editor de texto no convencional, tiene varios de los
> mismos plugins que Atom, VS Code, Sublime Text, etc. los cuales no serán
> cubiertos en este post, sólo se considerarán los que son específicos de Vim.

# Funcionalidades integradas

Al utilizar Vim, nuestra interacción con el mouse es innecesaria, dado que al
ser un editor modal, la mayoría del tiempo estás navegando, borrando o moviendo
texto más que escribiendo código. Sin entrar mucho en detalle, explicaré
brevemente cómo funciona Vim.

Éste editor consta de 3 modos básicos, donde sólo uno de ellos puede estar
activado a la vez, de esta forma podemos navegar, seleccionar, modificar e
insertar textos:

- **Normal Mode**: Todo lo que tipeamos en _Normal Mode_ es considerado un comando
  y mediante éstos podemos movernos, navegar a través del código (por ej.
  utilizando las teclas 'h', 'j', 'k' y 'l'), además, podemos borrar, copiar,
  mover textos, etc. La mayor parte del tiempo cuando ocupamos Vim estamos en
  este modo, supongo que de ahi viene el nombre

- **Insert Mode**: Utilizamos este modo para agregar texto dentro del buffer,
  como si estuviesemos ocupando cualquier otro editor de texto

- **Visual Mode**: Esto es _normal mode_ con la diferencia que los comandos de
  moviemiento extienden una área de selección, permitiendonos seleccionar texto
  sin la ayuda del mouse

Ahora que ya tenemos más o menos la idea de que existen al menos 3 distintos
modos en Vim, les mostraré comandos básicos que se utilizan estando en
_"Normal Mode"_.

## vim-surround

Este plugin "añade" un nuevo "_text object_" a Vim, lo que nos permite
interactuar con pares de caracteres envolventes, como paréntesis, tags HTML,
comillas, etc. como si fuese nativo de este editor.

> This plugin is a tool for dealing with pairs of “surroundings.” Examples of
> surroundings include parentheses, quotes, and HTML tags. They are closely
> related to what Vim refers to as |text-objects|. Provided are mappings to
> allow for removing, changing, and adding surroundings.
